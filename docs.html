Key Design Decisions

Server-Side Aggregations

All analytics, filtering, and pagination are computed in MongoDB using aggregation pipelines — not in JavaScript after data arrives. Dashboard stats run 8 parallel aggregations in a single request.

Feature-Sliced Client

Each product domain lives in its own features/ folder. Shared infrastructure lives in components/, hooks/, and services/. Prevents cross-feature coupling.

Data Layer

MongoDB Schemas

Auth credentials, role, 8 permission flags, avatar, status

Name, SKU, HSN code, GST rate, stockMatrix Map<color, Map<size, qty>>, images[]

Sequential number (JB-1001+), customer snapshot, items[], CGST/SGST, paid/due amounts, status
Customer
Name, phone, email, city, clientType (Retail/Wholesale/Boutique), status
Expense
Title, amount, category (10 types), payment mode, date, notes, receipt URL
AuditLog
userId, userName, action, details, timestamp — 90-day TTL auto-expiry
Counter
Atomic sequential bill number generator — prevents race condition duplicates
Settings
Store name, logo, GSTIN, address, invoice template preferences
Security

Authentication Flow

First-user rule. The first registered account automatically becomes the sole admin. Only one admin is allowed at a time. All subsequent registrations become members with configurable permission overrides.
Token Lifecycle
POST /auth/login
  → issues:  access token (15min, response body)
  → sets:    refresh token (7d, HttpOnly cookie)

Client attaches access token via Axios interceptor

On 401:
  → POST /auth/refresh (reads HttpOnly cookie)
  → new access token + rotated refresh token
  → original request retried transparently

POST /auth/logout
  → clears HttpOnly cookie server-side

API Reference

Auth

MethodRouteAuthDescription
POST/auth/registerPublicRegister. First user becomes admin.
POST/auth/loginPublicReturns access token + sets refresh cookie.
POST/auth/refreshCookieIssues new access token. Rotates refresh token.
POST/auth/logoutPublicClears HttpOnly refresh cookie.
GET/auth/meBearerReturns current user with permissions.

Products

MethodRouteAuthDescription
GET/productsBearerPaginated. Filters: search, category, stock, page, limit.
POST/productsPermissionCreate with variant matrix.
GET/products/:idBearerSingle product with computed totalStock.
PUT/products/:idPermissionUpdate and rebuild stock matrix.
PATCH/products/:id/stockBearerSet absolute variant stock quantity.
PATCH/products/:id/stock/adjustBearerAdd or subtract delta from variant stock.
DELETE/products/:idPermissionDelete product.

Billing

MethodRouteAuthDescription
GET/billsBearerPaginated. Filters: status, payment, from, to, search.
POST/billsBearerCreate bill. Auto-deducts stock. Auto-creates customer.
GET/bills/:idBearerSingle bill with all fields.
PATCH/bills/:id/statusBearerUpdate status. Cancelling restores stock.
GET/bills/export/csvPermissionDownload bills as CSV.

Customers

MethodRouteAuthDescription
GET/customersBearerAll customers with live stats. Single aggregation — no N+1.
GET/customers/:idBearerCustomer detail with ledger (last 20 bills).
POST/customersPermissionCreate customer record.
PUT/customers/:idPermissionUpdate customer details.
GET/customers/searchBearerTypeahead for billing autocomplete.

Expenses

MethodRouteAuthDescription
GET/expensesBearerPaginated with summary stats. Filters: category, from, to, search.
POST/expensesAdminRecord expense. Logged to audit trail.
PUT/expenses/:idAdminUpdate expense.
DELETE/expenses/:idAdminDelete expense. Logged to audit trail.
GET/expenses/summaryBearer6-month monthly expense totals for reports.

Dashboard

MethodRouteAuthDescription
GET/dashboard/statsBearer8 parallel aggregations: KPIs, revenue, payment mix, top products, low stock.
GET/dashboard/reportBearerP&L report. Params: period (daily/weekly/monthly), from, to.

Setup

Getting Started

1. Install dependencies
cd server && npm install
cd ../client && npm install
2. server/.env
MONGODB_URI=mongodb+srv://...
JWT_SECRET=<long-random-string>
JWT_EXPIRES_IN=15m
JWT_REFRESH_SECRET=<different-long-random-string>
JWT_REFRESH_EXPIRES_IN=7d
CORS_ORIGIN=http://localhost:5173
CLOUDINARY_CLOUD_NAME=...
CLOUDINARY_API_KEY=...
CLOUDINARY_API_SECRET=...
3. Run dev servers
# Terminal 1
cd server && npm run dev

# Terminal 2
cd client && npm run dev

# Client: http://localhost:5173
# API:    http://localhost:5000/api/v1
First login. Register the first account — it automatically receives admin privileges. All subsequent registrations are member accounts. Use User Management to configure per-member permissions.
